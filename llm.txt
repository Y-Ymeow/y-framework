# ymeow/php-framework

A modern, lightweight, Anti-MVC, route-first, component-driven PHP framework.

## Core Philosophy
- **Route-First**: Routes are defined using PHP 8 Attributes directly on controllers.
- **Component-Driven**: Server-driven responsive UI via `LiveComponent`.
- **Lightweight**: Minimal dependencies, optimized for performance and developer experience.

## Technical Stack
- **Language**: PHP 8.4+
- **DI Container**: Custom `Framework\Foundation\Container`
- **HTTP Layer**: Custom `Framework\Http\Request` / `Framework\Http\Response`
- **CLI**: Symfony Console
- **Frontend**: Vite + Idiomorph (for DOM patching)
- **Styling**: Built-in CSS Engine (Tailwind-like utility classes)

## Project Structure
- `src/`: Framework core logic.
- `app/`: Application-specific logic (Controllers, Components, Jobs).
- `admin/`: Admin-specific logic and pages.
- `config/`: Configuration files.
- `lang/`: Translation files (e.g. `lang/zh/messages.php`).
- `public/`: Web server root directory.
- `storage/`: Cache, logs, and temporary files.
- `database/migrations/`: Database migration files.
- `database/seeders/`: Database seeder files.
- `docs/`: Project documentation.

## Key Architectural Patterns
### 1. Routing
- Uses attributes like `#[Route('/path', methods: ['GET'])]`.
- Supports route groups via `#[RouteGroup('/prefix')]`.
- Supports middleware via `#[Middleware(['auth'])]`.
- Routing is scanned and can be cached using `php bin/console route:cache`.

### 2. LiveComponents
- Server-driven components that handle their own state and actions.
- State is compressed and base64 encoded: `base64(binary_sig + gzcompress(serialize(data)))`.
- Lifecycle: `boot()` → `mount()` → `hydrate()` → actions → `dehydrate()`.
- Actions use `#[LiveAction]` attribute and return `LiveResponse`.
- Fragment updates via `->liveFragment('name')` in render and `->fragment('name', $html)` in response.

### 3. Lifecycle & Hooks
- Implements an Action/Filter system similar to WordPress.
- Core hooks: `app.booting`, `app.booted`, `request.received`, `response.created`, `response.sending`, `response.sent`.
- Listeners are registered via `#[HookListener('hook.name')]`.

### 4. Database & ORM
- Custom QueryBuilder and Model implementation.
- Migration system for schema versioning.
- Seeder system for initial data.
- Multi-language support via `HasTranslations` trait.

## Common CLI Commands
- `php bin/console route:list`: List all registered routes.
- `php bin/console route:cache`: Generate route cache.
- `php bin/console cache:clear [type]`: Clear cache (route/config/view/all).
- `php bin/console migrate`: Run pending migrations.
- `php bin/console migrate:rollback`: Rollback last migration batch.
- `php bin/console db:seed [class]`: Run database seeders.
- `php bin/console make:page {Name}`: Create a new page.
- `php bin/console make:component {Name}`: Create a new LiveComponent.
- `php bin/console make:model {Name} [-m]`: Create a new Model (with optional migration).
- `php bin/console make:migration {name} [table]`: Create a migration.
- `php bin/console make:seed {Name}`: Create a seeder.
- `php bin/console make:config {name}`: Create a config file.
- `php bin/console make:translation {locale} [domain]`: Create a translation file.
- `php bin/console make:middleware {Name}`: Create a middleware.
- `php bin/console make:admin-resource {Name}`: Create an admin resource.
- `php bin/console key:generate`: Generate APP_KEY.
- `php bin/console css:generate`: Generate CSS from utility classes.

## Essential Helpers
- `app()`: Get the Application instance.
- `config('key')`: Get configuration value.
- `db()`: Get database connection.
- `auth()`: Get authentication manager.
- `user()`: Get current authenticated user.
- `redirect('/path')`: Return a redirect response.
- `abort(code, message)`: Throw an HTTP exception.
- `base_path('path')`: Get base path.
- `storage_path('path')`: Get storage path.
- `__('key')`: Get translation.
