# Plan: Fix Live Component Public State Integrity Check

## Problem Statement
The LiveComponent's public state integrity check is too strict. The `data-state` attribute (which contains public component properties as JSON) can be modified by frontend JavaScript, and when modified, the integrity check fails with "Live public state integrity check failed. Data tampering detected."

The current implementation:
1. Serializes public properties to `data-state` attribute as JSON
2. Computes an MD5 checksum of these properties and stores it in the server-side state
3. On subsequent requests, recomputes the checksum from received data and compares
4. If checksums don't match → tampering error

This causes false positives when:
- Frontend legitimately modifies state via JavaScript
- Frontend adds temporary UI state not part of component's public API
- Data types differ (e.g., number vs string) but semantically equivalent

## Root Cause Analysis

In `LiveComponent::fillPublicProperties()` (lines 561-599):
- The method receives `$data` from frontend (via `_data` parameter)
- Computes checksum of received data (filtered to only properties that exist in PHP class)
- Compares with stored `stateChecksum`
- If mismatch → throws RuntimeException

The issue: The checksum verification happens BEFORE applying changes, and it's too strict about exact matches.

## Solution Approach

### Option 1: Relax Checksum Verification (RECOMMENDED)
Modify the integrity check to:
1. Only verify properties that are explicitly part of the component's public API
2. Ignore extra properties sent by frontend (don't include them in checksum)
3. Allow type coercion for numeric/string equivalence
4. Add a configurable strictness level

### Option 2: Separate State and UI State
- Keep `data-state` for component public properties (verified)
- Use separate mechanism for frontend-only UI state (not verified)

### Option 3: Versioned State Snapshots
- Store state version/timestamp
- Allow updates as long as base version matches
- More complex but more flexible

## Implementation Plan

### Phase 1: Modify LiveComponent (Core Fix)

**File:** `src/Component/Live/LiveComponent.php`

1. **Add configuration option** for strictness level:
   - `strict` (default: true) - full checksum verification
   - `lenient` - only verify properties that exist in component, ignore extras
   - `coercive` - allow type coercion for numbers/strings

2. **Modify `generateDataChecksum()`** to support different modes:
   - In strict mode: current behavior
   - In lenient mode: only include properties that exist in component class
   - In coercive mode: normalize types before checksum

3. **Modify `fillPublicProperties()`** to:
   - Filter received data to only properties that exist in component
   - Apply type casting before checksum verification in coercive mode
   - Log warnings instead of throwing exceptions in lenient mode

4. **Add method** `setStateVerificationMode(string $mode)` to allow per-component configuration

## Code Changes

### LiveComponent.php Changes

```php
// Add property
protected string $stateVerificationMode = 'strict'; // strict|lenient|coercive

// Add setter
public function setStateVerificationMode(string $mode): void
{
    $this->stateVerificationMode = $mode;
}

// Modify generateDataChecksum to support modes
private function generateDataChecksum(array $data): string
{
    switch ($this->stateVerificationMode) {
        case 'lenient':
            // Only include properties that exist in component
            $ref = new \ReflectionClass($this);
            $publicProps = array_flip($this->allowedStateProperties());
            $data = array_intersect_key($data, $publicProps);
            break;
        case 'coercive':
            // Normalize types
            $data = $this->normalizeDataTypes($data);
            break;
    }
    
    $this->recursiveNormalize($data);
    return md5(json_encode($data, JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES | JSON_PRESERVE_ZERO_FRACTION));
}

// Modify fillPublicProperties to use mode
public function fillPublicProperties(array $data): void
{
    // ... existing code ...
    
    if ($this->stateChecksum !== null) {
        $dataToVerify = [];
        foreach ($publicProps as $propName) {
            if (array_key_exists($propName, $data)) {
                $dataToVerify[$propName] = $data[$propName];
            }
        }
        
        // Apply mode-specific normalization
        if ($this->stateVerificationMode === 'coercive') {
            $dataToVerify = $this->normalizeDataTypes($dataToVerify);
        }
        
        $currentChecksum = $this->generateDataChecksum($dataToVerify);
        
        if (!hash_equals($this->stateChecksum, $currentChecksum)) {
            if ($this->stateVerificationMode === 'lenient') {
                // Log warning but don't throw
                error_log("State checksum mismatch (lenient mode): {$this->stateChecksum} vs {$currentChecksum}");
            } else {
                throw new \RuntimeException('Live public state integrity check failed. Data tampering detected.');
            }
        }
    }
    
    // ... rest of method ...
}
```
